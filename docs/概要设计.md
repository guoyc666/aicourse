# AI 辅助的个性化学习系统 — 概要设计说明书

## 一、设计目标

本系统旨在构建一个结合 **知识图谱（Knowledge Graph）**、**大语言模型（LLM）** 与 **RAG（检索增强生成）** 技术的智能学习平台。  
系统通过自动解析教学资源、构建知识图谱、个性化学习路径推荐与智能问答，实现从“资源 → 知识 → 学习 → 反馈”的完整闭环。

系统支持三类用户：

- **教师**：上传课程资源、维护知识结构、查看学生学习状态；
- **学生**：学习课程内容、提问 AI 助教、参与测评并获得个性化推荐；
- **管理员**：维护系统数据与用户权限。

---

## 二、系统总体结构

系统整体采用 **分层架构设计**，以“知识驱动 + 模型增强”为核心逻辑。

```mermaid
graph TD
    A[教师上传资源] --> B[资源解析与知识点抽取]
    B --> C[知识图谱更新]
    C --> D[课程知识库与向量数据库]
    D --> E[垂直大模型与RAG问答]
    E --> F[AI助教回答与学习建议]
    F --> G[学生学习与答题记录]
    G --> H[学习画像与掌握度更新]
    H --> I[个性化学习路径推荐]
    I --> J[可视化学习反馈展示]
    J --> C
    subgraph 循环闭环
    C --> D
    D --> E
    end
```

该系统形成一个持续更新的**学习反馈闭环**：  
资源输入 → 知识结构化 → 模型生成 → 学习反馈 → 图谱更新。

## 三、系统主要功能流程说明

### 3.1 教师与系统的交互流程

```mermaid
sequenceDiagram
    participant 教师
    participant 系统
    participant 图谱
    participant 模型

    教师->>系统: 上传课程资源（PDF/PPT/视频）
    系统->>系统: 解析内容（OCR/NLP）
    系统->>图谱: 抽取知识点并更新节点
    图谱->>模型: 提供知识库数据
    教师->>系统: 查看知识图谱结构
    教师->>系统: 查看学生学习进度与答题结果
```

---

### 3.2 学生与系统的学习活动流程

```mermaid
sequenceDiagram
    participant 学生
    participant 系统
    participant 向量库
    participant 模型
    participant 图谱

    学生->>系统: 登录系统
    学生->>系统: 学习资源 / 浏览学习路径
    学生->>系统: 向AI助教提问
    系统->>向量库: 检索相关知识内容（RAG）
    向量库-->>系统: 返回增强上下文
    系统->>模型: 输入增强后的Prompt
    模型-->>系统: 生成个性化回答
    系统-->>学生: 返回回答与学习建议
    学生->>系统: 完成测验题目
    系统->>图谱: 更新掌握度与学习记录
    图谱-->>系统: 生成个性化学习路径
    系统-->>学生: 展示学习可视化图谱
```

## 四、系统核心逻辑流与数据流

```mermaid
flowchart LR
    A[上传资源] --> B[解析提取文本/字幕/关键字]
    B --> C[知识点抽取与命名实体识别]
    C --> D[知识图谱节点与关系生成]
    D --> E[存入Neo4j]
    E --> F[构建向量数据库（FAISS）]
    F --> G[模型微调与RAG检索]
    G --> H[AI问答与路径推荐]
    H --> I[学生学习记录与掌握度更新]
    I --> J[知识图谱更新与可视化]
```

## 七、系统特性与总结

- 知识驱动：以知识图谱为核心结构；
- 智能增强：基于 RAG + 大模型实现智能问答；
- 个性化推荐：结合学习路径与掌握度；
- 多模态支持：文本、视频、PPT；
- 可视化交互：学习图谱与进度展示；
- 可扩展架构：适合课程项目与后续优化。
